## 🎯 v0.1 MVP（第一周）

### 目标
验证核心假设：用户愿意和AI聊天获得情绪支持

### 功能清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 对话界面 | 简单的聊天页面 | P0 |
| AI回复 | 基于工作流手册的温暖回应 | P0 |
| 对话历史 | 浏览器本地存储 | P1 |

### 不做
- 用户登录
- 数据库
- 情绪识别
- 风险预警

### 交付物
- 可访问的URL
- 5个测试用户

### 执行步骤

```
Day 1-2: 创建简化版后端
  ├── 使用简单的方式调用LLM
  └── 创建基础对话API

Day 3-4: 创建前端
  ├── 聊天界面
  ├── 消息列表
  └── 发送/接收消息

Day 5: 部署
  ├── 选择部署平台
  ├── 配置自动部署
  └── 分享URL

Day 6-7: 测试
  ├── 找5个用户测试
  └── 收集反馈
```

## 🔧 技术选型

### 当前版本（v0.1-v0.3）

| 组件 | 技术 | 原因 |
|------|------|------|
| 前端 | React + Vite | 快速开发 |
| 后端 | FastAPI | Python生态 |
| LLM | 暂定 | 根据效果选择 |
| 部署 | Vercel + Railway | 免费额度 |
| 数据库 | SQLite (MVP) -> PostgreSQL | 简单到生产级 |

### 未来版本（v2.0+）

| 组件 | 技术 | 原因 |
|------|------|------|
| LLM | OpenAI / Anthropic | 稳定性 |
| 数据库 | PostgreSQL + Redis | 生产级 |
| 存储 | S3 | 文件存储 |
| 支付 | Stripe | 国际支付 |

## ⚠️ 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| LLM成本 | 费用超支 | 限制调用次数，优化prompt |
| 用户流失 | 无反馈 | 主动联系用户 |
| 风险漏报 | 用户安全 | 保守策略，宁误报不漏报 |
| 合规问题 | 法律风险 | 明确免责声明 |

## 📞 快速开始命令

```bash
# 克隆项目
git clone https://github.com/121212165/keyi.git
cd keyi

# MVP开发
# 1. 启动后端
cd backend
python run.py

# 2. 启动前端（新终端）
cd frontend
npm run dev

# 3. 访问 http://localhost:5173
```
